    • Evaluation of Multiple Classification algorithms in the task of Facial Expression Recognition.

    Comparison of classification algorithms for the task of Facial Expression Recognition. Also includes real-time Facial Expression Recognition via screen capture.(LinearSVM+RbfSVM+MLP+AdaBoost+NBayes).

    A  User Manual

     

    Pre-requisites

    In order to use the system, the following is required:

    1.    Visual Studio 15 or newer

    2.    OpenCV 3.0 or newer

    3.    Dlib 18.18 or newer

    4.    Boost 1.58.0 or newer

    5.    CK+ dataset

    6.    Windows 7 or newer

    The system might run with older versions of said software, however, that is not guaranteed.

     

    Using the System

    To use the pre-compiled system, just run the executable. A console window will open up and you will be asked whether you would like to load the pre-processed dataset. The pre-processed dataset is stored in save.txt in the same folder as the executable.

    Dataset

    The dataset itself resides in .\dataset\ and follows a specific directory structure. Firstly, it contains 2 folders – images and labels. Each of those contains a list of corresponding folders named S*** where *** is the number of the subject. Each subject (S***) folder contains folders containing image sequences or a label of a particular expression (e.g. \dataset\images\S032\005\ S032_005_00000001.png through to \dataset\images\S032\005\S032_005_00000015.png whereas the corresponding label is stored in \dataset\labels\S032\005\S032_005_00000016_emotion.txt).

    The CK+ dataset was used for the development of this software, which can be obtained from http://www.consortium.ri.cmu.edu/ckagree/

    Classification models

    After the dataset has been loaded, the software will prompt you for each of the classification algorithms whether you would like to train the model or load it from the corresponding XML file (stored in the same folder as the executable).

    Live testing

    Once everything is loaded a window with a transparent background will pop up. This window acts as a viewfinder. First, you have to select a neutral face and press the “N” key.

    After that you can select an image containing a facial expression and press “Spacebar”. The result will be displayed in the console.

    Alongside the console output the both pre-processed faces will be visible in separate windows, as well as, overlapped.

    Set-up

    To compile the project you will need a machine running Windows 7 or newer (SetLayeredWindowAttributes() used inside the GUI class might behave differently on versions of Windows prior to 7). Additionally you will need to have Visual Studio 13 or newer (VS15 was used for the development of this project). After which you will need to set-up all of the required dependencies – OpenCV, Dlib and Boost.

     

    OpenCV

    1.    Download the latest version of OpenCV from the official website (opencv.org) and extract it somewhere.

    2.    In Visual Studio open the Property Manager (View ‣ Property Pages) for the project you would like to add OpenCV to.

    3.    Right click the build configuration you would like to use and click on “Add New Project Property Sheet”

     


    4.    Click the “C/C++ > General” entry and under the “Additional Include Directories” add the path to your OpenCV include.

     

    5.    Next go to the “Linker > General” and under the “Additional Library Directories” add the libs directory

     

     

     

    6.    Next go to “Linker > Input” and under “Additional Dependencies” specify the modules to use. For OpenCV 3.0+ only “opencv_world[version][d].lib” (replace [version] with 300 for OpenCV 3.0 or 310 for 3.1.0; the [d] after [version] should be included for debug configurations and omitted for release – e.g opencv_world300d.lib for debug and opencv_world300.lib for release) should be included.

    For more detailed information, please refer to the OpenCV website - http://docs.opencv.org/2.4/doc/tutorials/introduction/windows_visual_studio_Opencv/windows_visual_studio_Opencv.html

    Summary

    After you have set-up all of the required libraries and have acquired the dataset or a savefile of a pre-processed version (save.txt). You should be able to compile and run the project.

    Porting to other OS

    The project was written with portability in mind. The only classes that are OS specific are – GUI and Capture and, as such would have to be rewritten. All of the libraries used support multiple operating systems and claim high adaptability. Here are some quotes from the library websites:

    “Boost works on almost any modern operating system, including UNIX and Windows variants.” – www.boost.org

    “The library is tested regularly on MS Windows, Linux, and Mac OS X systems. However, it should work on any POSIX system and has been used on Solaris, HPUX, and the BSDs.” – www.dlib.net

    “It has C++, C, Python and Java interfaces and supports Windows, Linux, Mac OS, iOS and Android.” – www.opencv.org

    Known bugs

    There are a number of bugs present in the system at this moment, which are yet to be addressed:

    ·         In the live testing part of the application if two screenshots of the same neutral face are taken but the dimensions of the screen capture window differ between the two – the images will not align perfectly. I have put in quite a bit of time into trying to resolve this and have come to the conclusion that Dlib’s facial landmark extraction algorithm will return slightly different landmark positions depending on the dimensions of the input image. However, this should have minimal effect on the overall performance of the system.

    ·         If you attempt to take a screenshot while the screen capture window is moved outside of the screen region the following assertion will be triggered “error: (-215) 0 <= roi.x && 0 <= roi.width && roi.x + roi.width <= m.cols && 0 <= roi.y && 0 <= roi.height && roi.y + roi.height <= m.rows in function cv::Mat::Mat”

     

    Future development

    ·         The main feature to be added in the future is automated facial expression recognition. That is, at this point the user has to select the neutral face and the face containing the facial expression themselves. In an automated version of this software both of those frames will have to be selected automatically. Which is a very difficult task in itself.

    ·         Another feature that could be added is the ability to work with multiple faces in the same image. The algorithm currently in place can be adjusted to detect multiple faces. After which those can be dealt with separately. The difficulty, however, lies in the fact that faces can move around. And since this algorithm requires a neutral face to be paired with the face showing an expression either some form of tracking needs to be in place or some form of facial recognition.
